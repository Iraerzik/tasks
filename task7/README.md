# Чат-сервер и клиент на языке C

## Описание реализации

Данный проект представляет собой простую систему чата, реализованную по модели "клиент-сервер" с использованием сокетов TCP. Сервер поддерживает одновременное подключение до 30 клиентов, обработку публичных и приватных сообщений, а также базовые команды управления.

### Особенности реализации:
- Многопользовательский чат с использованием мультиплексирования ввода-вывода (poll)
- Поддержка публичных сообщений (отправляются всем участникам)
- Поддержка приватных сообщений между пользователями
- История приватных контактов для каждого пользователя
- Система команд с префиксом '\'
- Обработка отключения клиентов и корректное завершение работы сервера

## Процесс сборки проекта

Для сборки проекта используется Makefile. Основные команды:

```bash
make        # сборка сервера и клиента
make clean  # очистка объектных файлов и исполняемых файлов
```

### Makefile:
```makefile
CC = gcc
CFLAGS = -Wall -Wextra -Werror -g
TARGETS = server client
HEADERS = chat.h

all: $(TARGETS)

server: server.c $(HEADERS)
	$(CC) $(CFLAGS) -o server server.c

client: client.c $(HEADERS)
	$(CC) $(CFLAGS) -o client client.c

clean:
	rm -f $(TARGETS) *.o

.PHONY: all clean
```

## Процесс запуска проекта

### Запуск сервера:
```bash
./server <порт>
```
Пример:
```bash
./server 8888
```

### Запуск клиента:
```bash
./client <хост> <порт>
```
Пример:
```bash
./client localhost 8888
```

### Процесс работы:
1. Запустите сервер на выбранном порту
2. Запустите один или несколько клиентов, указав хост и порт сервера
3. При подключении клиент вводит свой никнейм
4. После подключения доступны команды чата

### Доступные команды:
- `\help` - показать список доступных команд
- `\users` - показать список онлайн пользователей
- `\private <ник> <сообщение>` - отправить приватное сообщение пользователю
- `\privates` - показать список контактов для приватных сообщений
- `\quit <сообщение>` - выйти из чата с прощальным сообщением

Любое сообщение, не начинающееся с '\', отправляется как публичное сообщение всем участникам чата.

## Структура проекта и содержимое файлов

### chat.h
Заголовочный файл, содержащий:
- Подключение необходимых библиотек
- Определение констант (MAX_CLIENTS, BUFFER_SIZE, NAME_SIZE и др.)
- Структуру Client, хранящую информацию о подключенном клиенте:
  - `fd` - файловый дескриптор сокета
  - `name` - никнейм клиента
  - `addr` - адрес клиента
  - `active` - флаг активности
  - `private_contacts` - массив никнеймов для истории приватных сообщений
  - `contact_count` - количество контактов

### server.c
Основной файл серверной части, реализующий:
- Инициализацию и управление клиентами
- Обработку новых подключений
- Маршрутизацию сообщений (публичные и приватные)
- Обработку команд клиентов
- Управление историей приватных контактов
- Обработку сигналов для корректного завершения

Основные функции:
- `main()` - главный цикл сервера с poll
- `handle_new_connection()` - обработка нового подключения
- `handle_command()` - обработка команд от клиента
- `send_msg()` - отправка публичных сообщений
- `send_private_message()` - отправка приватных сообщений
- `remove_client()` - удаление отключившегося клиента

### client.c
Файл клиентской части, реализующий:
- Подключение к серверу
- Ввод ника при подключении
- Дуплексную передачу сообщений (отправка с клавиатуры и прием от сервера)
- Использование select для одновременного отслеживания ввода с клавиатуры и данных от сервера

## Используемые технологии

Проект реализован на языке C с использованием только стандартных библиотек:

- **Сетевые взаимодействия**: сокеты TCP (AF_INET, SOCK_STREAM)
- **Мультиплексирование ввода-вывода**: poll (сервер), select (клиент)
- **Обработка сигналов**: signal (для корректного завершения сервера)
- **Стандартные библиотеки**: stdio.h, stdlib.h, string.h, unistd.h и др.

Сторонние библиотеки не используются.

### Известные ограничения:
- Максимальная длина сообщения - 1024 байта
- Максимальная длина ника - 32 символа
- Максимальное количество клиентов - 30
- История приватных контактов хранится только в оперативной памяти и теряется при перезапуске сервера

### Тестирование:
Проект протестирован в среде Linux. Для компиляции и работы требуется ОС семейства Unix (Linux, macOS). На Windows рекомендуется использовать WSL или виртуальную машину.
